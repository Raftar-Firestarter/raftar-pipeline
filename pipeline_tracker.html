Firebase Setup Guide
One-time 5-minute setup. Free forever for your team size.
Step 1 — Create a Firebase project
Go to console.firebase.google.com and sign in with your Google account.
Click "Add project" → name it raftar-pipeline → disable Google Analytics (not needed) → click "Create project".
Step 2 — Create a Realtime Database
In the left sidebar click Build → Realtime Database.
Click "Create Database" → choose a location (any is fine) → select "Start in test mode" → click "Enable".
Step 3 — Register a Web App
In the left sidebar click the ⚙ gear icon → Project settings.
Scroll down to "Your apps" → click the </> Web icon.
Give it a nickname (e.g. pipeline-tracker) → click "Register app".
Step 4 — Paste your config into this file
Firebase will show you a firebaseConfig object that looks like this:

const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "raftar-pipeline.firebaseapp.com",
  databaseURL: "https://raftar-pipeline-default-rtdb.firebaseio.com",
  projectId: "raftar-pipeline",
  storageBucket: "raftar-pipeline.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abc123"
};

Open this HTML file in a text editor (Notepad, VS Code, etc.) and find the section at the top that says REPLACE THESE 3 VALUES. Replace each placeholder with your actual values.
Step 5 — Host it on GitHub Pages (free)
1. Go to github.com → sign up / sign in → click "New repository"
2. Name it raftar-pipeline → set to Public → click "Create repository"
3. Click "uploading an existing file" → drag this HTML file → click "Commit changes"
4. Go to Settings → Pages → under Source select main branch → click Save
5. GitHub gives you a link like: https://yourusername.github.io/raftar-pipeline/pipeline_tracker.html

Share that link with your team. Anyone who opens it will see the same live board.
Step 6 — Set Database Rules (keep it open)
In Firebase console go to Realtime Database → Rules and set:

{
  "rules": {
    ".read": true,
    ".write": true
  }
}

Click Publish. This allows anyone with the link to read and write — which is what you want.
Note: If you ever want to lock it down later, you can add authentication. But this is fine to start with.
✓ Done? Here's how to verify it's working
1. Open the hosted link on your phone
2. Open it on your laptop at the same time
3. Add a card on one device — it should appear on the other within 1–2 seconds
4. The green dot in the top-left should say "Live" instead of "Demo mode"
